bm (n=1, reps=5, &blk)

Run a benchmark the healthy way: with a warmup run, with multiple repetitions, and standard deviation.

  • n Number of times to execute blk (one data sample)
  • reps Number of data samples to collect
  • blk a Ruby block to benchmark

Returns a Benelux::Tms object

[show source]

240:   end
current_track (name=nil,group=nil)

If name is specified, this will associate the current thread with that Track name (the Track will be created if necessary).

If track is nil, it returns the Track object for the Track associated to the current thread.

update_global_timeline ()

Only updates data from threads that are dead and rotated timelines.

[show source] 
     # File lib/benelux.rb, line 88
 88:   def Benelux.update_global_timeline
 89:     @@mutex.synchronize do
 90:       dthreads = Benelux.known_threads.select { |t| 
 91:         !t.timeline.nil? && (t.nil? || !t.status) &&
 92:         !@processed_dead_threads.member?(t)
 93:       }
 94:       # Threads that have rotated timelines
 95:       rthreads = Benelux.known_threads.select { |t|
 96:         !t.rotated_timelines.empty?
 97:       }
 98:       dtimelines = dthreads.collect { |t| t.timeline }
 99:       # Also get all rotated timelines. 
100:       rthreads.each { |t| 
101:         # We loop carefully through the rotated timelines
102:         # incase something was added while we're working. 
103:         while !t.rotated_timelines.empty?
104:           dtimelines.push t.rotated_timelines.shift 
105:         end
106:       }
107:       Benelux.ld [:update_global_timeline, dthreads.size, rthreads.size, dtimelines.size].inspect
108:       # Keep track of this update separately
109:       @timeline_chunk = Benelux::Timeline.new
110:       @timeline_chunk.merge! *dtimelines
111:       @processed_dead_threads.push *dthreads
112:       tl = Benelux.timeline.merge! Benelux.timeline_chunk
113:       @timeline_updates += 1
114:       tl
115:     end
116:   end
